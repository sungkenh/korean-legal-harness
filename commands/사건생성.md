# 사건 폴더 생성

입력: **$ARGUMENTS** (예: `2026가단12345 홍길동v김철수`)

## 동작

1. `$ARGUMENTS`를 공백으로 분리해 사건번호·당사자약칭 추출.
2. 폴더명 생성: `{사건번호}-{당사자약칭}` (공백 허용 안 됨 — 하이픈·언더스코어로 치환).
3. 아래 경로를 생성:
   ```
   notes/법무/사건/{폴더명}/
   ├── 사건개요.md      (템플릿 채움)
   ├── 관련법령.md      (빈 템플릿)
   ├── 증거목록.md      (빈 템플릿)
   ├── 일정.md          (빈 템플릿)
   ├── 서면/.gitkeep
   ├── 자료/.gitkeep
   └── 통신/.gitkeep
   ```
4. 사건번호 미부여 의뢰 단계: `의뢰-{YYYYMMDD}-{약칭}` 형식 사용 (사용자에게 먼저 확인).

## 템플릿

### 사건개요.md
```markdown
# 사건개요

## 기본정보
- 사건번호: {사건번호}
- 법원·재판부: 
- 담당변호사: 
- 사건분류: 
- 의뢰일: {오늘 날짜 YYYY-MM-DD}

## 당사자
- 원고/신청인: 
- 피고/피신청인: 
- 대리인: 

## 청구취지·신청취지
1. 

## 사실관계 요약
(시간순 3~5문단)

## 쟁점
1. 
2. 

## 현황·진행상황
- {오늘 날짜}: 사건 폴더 생성

## 관련 파일
(자료/, 서면/, 통신/ 하위에 추가되는 파일 링크)
```

### 관련법령.md
```markdown
# 관련 법령·판례

> `/리서치` 커맨드 실행 결과가 여기에 누적 기록됩니다. 기존 내용 덮어쓰기 금지.

## (첫 리서치 결과가 YYYY-MM-DD 헤더 아래 추가됩니다)
```

```markdown
| 번호 | 명칭 | 입증취지 | 비고 |
```

### 일정.md
```markdown
# 기일·제출기한

| 일자 | 구분 | 내용 |
|---|---|---|
```
