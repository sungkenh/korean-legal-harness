# 계약서 검토

대상: **$ARGUMENTS**

## 동작

1. 인자 파싱: 파일 경로 + 선택적 의뢰인 입장.
   - 입장 미지정 시 사용자에게 먼저 질문 ("의뢰인은 甲이신가요, 乙이신가요?").
2. 파일 유형 확인 (PDF/DOCX/MD/HTML).
3. `Agent(subagent_type="contract-reviewer-kr")` 호출:
   - 입력: 파일 경로, 의뢰인 입장.
4. 검토 보고서를 표시하고 `{파일경로}.review.md`로 저장 제안.

## 후속 파이프라인 (선택)

- **수정안 반영본 작성** — 사용자 요청 시 수정안을 반영한 새 파일 생성.
- **협상용 커버레터 작성** — `/초안` 파이프라인 변형 호출.
- **유사 계약 판례 리서치** — `/리서치` 연계.

## 실패 처리

- 파일이 이미지 PDF면 `anthropic-skills:pdf` 스킬 선행 실행.
- 100매 초과 대형 계약은 챕터 단위 분할 검토 제안.
