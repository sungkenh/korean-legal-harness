# 법률서면 롱폼 초안 파이프라인

입력: **$ARGUMENTS** (예: `답변서 2026가단12345`, `항소이유서 notes/법무/사건/...`)

## 지원 서면 유형

- 준비서면, 답변서, 고소장, 고발장, 항소이유서, 상고이유서, 항고이유서, 의견서, 변론요지서

## 사전 조건

1. 사건 폴더가 존재해야 함. 없으면 `/사건생성`을 먼저 실행하도록 안내.
2. `사건개요.md`에 기본정보·당사자·청구취지·사실관계가 채워져 있어야 함. 비어 있으면 작성 유도.

## 워크플로우

### Step 1. 사건 컨텍스트 수집
- `사건개요.md`, `관련법령.md`, `증거목록.md` 읽기.
- `자료/` 내 핵심 파일 식별(Glob).
- 리서치 부족 시 사용자에게 `/리서치`를 먼저 실행할지 확인.

### Step 2. 개요 생성 (legal-longform-drafter)
`Agent(subagent_type="legal-longform-drafter")`에 위임:
- 지시: "**개요만** 작성. 서면 유형={유형}, 사건폴더={경로}. 논리구조·인용예정·핵심주장·분량 제시."

### Step 3. 사용자 승인
개요를 보여주고 **승인 또는 수정안을 받는다**. 승인 전 본문 작성 금지.

### Step 4. 본문 작성 (legal-longform-drafter 재호출)
승인된 개요로 본문 작성. 저장 경로: `{사건폴더}/서면/{NN-유형}.md` (NN은 기존 서면 수 + 1).

### Step 5. 인용 검증 (citation-checker-kr)
`Agent(subagent_type="citation-checker-kr")` 호출 → 오류 보고 → 사용자 승인 시 수정 반영.

### Step 6. 교열 (legal-editor-kr)
`Agent(subagent_type="legal-editor-kr")` 호출 → 경미 이슈는 자동 적용, 중대 이슈는 사용자 승인 후 적용.

### Step 7. 최종 보고
```markdown
초안 완성: {서면유형}

- 파일: {경로}
- 분량: A4 N매 (M자)
- 인용: 법령 N건 / 판례 N건 (검증 통과)
- 교열: 경미 N건 적용, 중대 N건 반영
- 다음 단계: 담당 변호사 검토 → DOCX/PDF 변환 → 법원 제출

⚠ 본 초안은 변호사 검토·서명 없이 법원에 제출할 수 없습니다.
```

## 실패 처리

- 사건개요 부실 → 작성 권유 후 중단
- 관련법령 부실 → `/리서치` 권유 후 중단
- 인용 치명 오류 다수 → 리서치 보강 후 재작성
