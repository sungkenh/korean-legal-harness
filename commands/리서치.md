# 법령·판례 풀 리서치

쟁점: **$ARGUMENTS**

## 워크플로우

### Step 1. 쟁점 명확화 (필요 시)
입력이 모호하면 한 번 되물어 법적 쟁점을 구체화. 예: "손해배상" → "계약해제 후 손해배상의 범위(이행이익 vs 신뢰이익)".

### Step 2. 리서치 실행 (legal-researcher-kr)
`Agent(subagent_type="legal-researcher-kr")`에 위임:
- 입력: `$ARGUMENTS`
- 지시: "`chain_full_research` 또는 `chain_dispute_prep` 우선. 법령 3개+판례 3~5개 수집. verify_citations로 자체 검증."

### Step 3. 결과 확인 및 저장
1. 에이전트 결과물을 사용자에게 보여준다.
2. 현재 대화 맥락에 사건 폴더가 있으면 (예: `/사건질문` 직후, 또는 사용자가 `사건/{폴더}` 언급) → 해당 `관련법령.md`에 **추가** (덮어쓰기 금지).
   - 헤더: `## YYYY-MM-DD — {쟁점}` 아래에 append.
3. 사건 폴더 맥락이 없으면 → `notes/법무/리서치/{날짜}-{쟁점슬러그}.md` 새 파일로 저장.

### Step 4. 후속 제안
- 서면 초안 필요 시: `/초안 <유형> <사건번호>` 안내.
- 상충 판례가 있으면 사용자에게 사안 적용 방향 문의.
- 추가 리서치 필요 영역 명시.

## 출력 요약 (사용자 가시)

```markdown
리서치 완료: **{쟁점}**

- 법령 N건 / 판례 N건 수집
- 자체 검증(verify_citations): 통과 N / 수정 N
- 저장 위치: {경로}
- 핵심 결론: {1~2문장}

**다음 제안**
- {후속 단계 1~2개}
```
