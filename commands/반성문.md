# 반성문·탄원서 작성

입력: **$ARGUMENTS**

## 사전 확인 (에이전트가 사용자에게 질문)

### 반성문
- 피고인 성함·사건번호·죄명
- 범행 경위 (객관적 사실)
- 범행 동기
- 피해자와의 합의·공탁 진행 상황
- 반성의 구체적 행동 (서신·치료·상담 등)
- 재범 방지 조치
- 가족·직업 상황

### 탄원서
- 탄원인 성함, 피고인과의 관계
- 피고인의 평소 인성·생활상
- 구체적 일화 (피고인의 선행·책임감)
- 선처 호소 근거

## 워크플로우

### Step 1. 입력 수집
`Agent(subagent_type="apology-writer-kr")`가 사용자에게 위 정보를 단계적으로 질문.

### Step 2. 초안 작성
수집된 정보로 반성문/탄원서 초안을 작성하여 아래 경로에 저장:
- 반성문: `{사건폴더}/서면/반성문-{YYYYMMDD}.md`
- 탄원서: `{사건폴더}/서면/탄원서-{탄원인}-{YYYYMMDD}.md`

### Step 3. 교열 (legal-editor-kr)
- 반성문은 **1인칭 진솔체** 유지 검증.
- 탄원서는 **3인칭 탄원인 시점** 유지.
- 금지어 자동 제거.

### Step 4. 최종 보고
- 저장 경로
- 분량
- 개인정보 포함 여부 (주민번호/연락처 마스킹 점검)
- 다음 단계: 의뢰인 친필 옮겨 적기 또는 서명.

## 주의

- AI가 생성한 초안은 **의뢰인이 직접 옮겨 쓰거나 수정**해야 진정성을 담을 수 있다. 재판부가 양산된 반성문 패턴을 인지할 수 있음을 사용자에게 안내.
- 합의·공탁이 실제 없는 경우 "있는 것처럼" 쓰지 말 것.
