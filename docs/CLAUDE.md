# 법무법인 업무 폴더 (자동 로드)

이 폴더에 진입하면 아래 지침이 자동으로 적용됩니다.

## 원칙

1. **한국 법률 실무 기준** — 모든 문서는 대한민국 법령·판례·실무 관행에 따른다.
2. **정확한 인용** — 법령은 조·항·호까지, 판례는 선고일·사건번호까지 명시한다. 추측 금지.
3. **출처 우선순위** — `korean-law-mcp`(법제처·대법원) > 공식 관보 > 학설·실무서적. AI의 기억 단독 인용 금지.
4. **존댓말·격식체** — 의뢰인·법원 제출 문서는 "~합니다", "~사료됩니다" 등 격식체. 내부 메모는 간결한 문어체 허용.
5. **개인정보 보호** — 주민등록번호, 연락처, 계좌번호 등은 마스킹(`****`) 또는 별지 처리. 저장 시 훅이 자동 경고.
6. **면책** — 최종 서면은 반드시 담당 변호사가 검토·서명. AI 결과는 초안이며 법률자문이 아님을 명시한다.
7. **한글(HWP) 한계** — 네이티브 한글 파일은 파싱 미지원. DOCX/PDF 변환 후 처리.

## 5대 기능축 (슈퍼로이어 대응)

| 축 | 담당 에이전트 | 커맨드 |
|---|---|---|
| ① 법률 리서치 | `legal-researcher-kr` | `/리서치` |
| ② 초안 작성(롱폼) | `legal-longform-drafter` | `/초안` |
| ③ 문서 요약 | `legal-summarizer-kr` | `/요약` |
| ④ 문서 기반 대화 | `document-qa-agent` | `/문서질문` |
| ⑤ 사건 기반 대화 | `case-folder-qa-agent` | `/사건질문` |

도메인 특화: `contract-reviewer-kr`(계약검토), `apology-writer-kr`(반성문), `citation-checker-kr`(인용검증), `legal-editor-kr`(교열).

## 디렉토리 구조

```
notes/법무/
├── CLAUDE.md           # 본 파일
├── docs/               # 가이드 문서
├── 사건/               # 사건별 폴더 (→ docs/사건-폴더-운영가이드.md)
│   └── {사건번호}-{당사자}/
│       ├── 사건개요.md
│       ├── 관련법령.md
│       ├── 증거목록.md
│       ├── 서면/
│       ├── 자료/
│       └── 통신/
├── 서식/               # 재사용 서면 템플릿
└── 자문/               # 법률자문서 아카이브
```

## 자주 쓰는 커맨드

- `/사건생성 <사건번호> <당사자약칭>` — 사건 폴더 + 템플릿 자동 생성
- `/리서치 <쟁점>` — 법령+판례 풀 리서치
- `/초안 <유형> <사건번호>` — 롱폼 서면 작성 파이프라인
- `/요약 <파일>` — 문서 요약 + 쟁점 추출
- `/문서질문 <파일> <질문>` — 단일 문서 Q&A
- `/사건질문 <사건폴더> <질문>` — 사건 폴더 통합 Q&A
- `/계약검토 <파일>` — 조항별 리스크 감사
- `/반성문 <사건>` — 양형자료 작성
- `/인용검증 <파일>` — 인용 오류 일괄 검증
