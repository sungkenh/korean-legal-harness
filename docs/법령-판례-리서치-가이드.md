# 법령·판례 리서치 가이드

## 사용 도구 (korean-law MCP)

### 단일 조회
- `search_law` — 법령 키워드 검색 (예: "임대차", "근로기준법")
- `get_law_text` — 특정 법령 본문 (법령ID/법령명)
- `search_decisions` — 판례 키워드 검색
- `get_decision_text` — 판례 본문 (사건번호)
- `get_annexes` — 별표·별지 서식
- `verify_citations` — 문서 내 인용 자동 검증

### 복합 체인 (우선 고려)
| 체인 | 용도 |
|---|---|
| `chain_full_research` | 쟁점 하나에 대한 법령+판례+해설 풀 리서치 |
| `chain_law_system` | 특정 법률의 상하위·관련 법령 체계도 |
| `chain_ordinance_compare` | 지자체 조례 비교 |
| `chain_action_basis` | 특정 행정처분의 법적 근거 찾기 |
| `chain_amendment_track` | 법령 개정 이력 추적 |
| `chain_dispute_prep` | 분쟁 대비 법령+판례 종합 |
| `chain_document_review` | 제출 문서의 법령 정합성 검토 |
| `chain_procedure_detail` | 특정 절차의 세부 단계 |

## 표준 리서치 플로우

1. **쟁점 분해** — 의뢰인 질의를 구성요건·효과 단위로 쪼갠다.
2. **법령 근거** — `search_law` → `get_law_text`로 해당 조문 확정.
3. **해석 판례** — `search_decisions`로 동일·유사 사실관계 판례 3~5건 수집.
4. **대법원 판례 우선** — 하급심은 참고용. 최신순·관련도순 정렬.
5. **인용 검증** — 초안 작성 후 `verify_citations`로 오기 일괄 점검.
6. **체계 확인** — 시행령·시행규칙까지 확인. 별표가 있으면 `get_annexes` 필수.

## 인용 시 주의

- 법령은 **시행일 기준 버전** 명시. 개정 전후 구분.
- 판례는 "대법원 2023. 5. 11. 선고 2022다12345 판결" 형식 준수 (인용-형식-규칙.md 참조).
- 판례 요지는 **원문 직접 인용**. 요약 시 "판례 취지"로 표기.
